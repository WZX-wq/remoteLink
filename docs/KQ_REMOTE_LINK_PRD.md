# KQremoteLink 产品需求文档

> 文档版本：v1.0
> 更新日期：2026-06-25
> 适用项目：KQremoteLink / 鲲穹远程桌面
> 当前阶段：商业化测试版到正式上线前整理

## 1. 项目概述

KQremoteLink 是基于 RustDesk 能力定制的鲲穹远程桌面产品，面向个人用户、售后支持、企业设备维护和跨设备办公场景，提供 Windows 桌面端、Android 移动端和项目服务器能力。

产品保留 RustDesk 的核心远程控制能力，包括设备 ID、密码连接、屏幕传输、鼠标键盘控制、剪贴板、文件传输、服务模式和私有中继服务器；在此基础上加入鲲穹账号登录、会员权益、设备列表、最近连接、移动端横屏全屏远控、支付、下载页、安装包发布和正式服务器迁移能力。

## 2. 产品目标

1. 用户可以通过鲲穹账号在多端登录，并看到同账号下可远控的设备。
2. 用户可以从 Windows 或 Android 发起远程连接，完成稳定的桌面控制。
3. 手机远控电脑时，默认进入横屏全屏体验，减少遮挡并接近向日葵类产品的操作习惯。
4. 用户可以查看最近连接、移动设备、桌面设备，并对本地最近连接执行刷新、重命名、删除、星标等操作。
5. 用户可以购买会员，并按会员权益解锁更高画质、分辨率、帧率和连接数量。
6. 项目可以通过自有服务器提供 hbbs/hbbr、项目 API、下载页、支付回调和安装包发布。
7. Windows 安装包和 Android APK 可以被持续构建、发布、验证，并展示正确版本号。

## 3. 用户角色

| 角色 | 说明 | 核心诉求 |
| --- | --- | --- |
| 普通远控用户 | 使用自己的手机或电脑远程控制另一台电脑 | 快速连接、操作顺手、画面清晰 |
| 被协助用户 | 临时让他人连接自己的电脑 | 找到 ID/密码、明确授权、安全可控 |
| 付费会员 | 需要更高画质、更多设备、更多连接记录 | 会员权益立即生效、支付流程顺畅 |
| 售后/运维人员 | 频繁连接客户设备或公司设备 | 最近连接可靠、设备列表清楚、连接稳定 |
| 管理/运营人员 | 负责部署、发布、下载页和服务器迁移 | 包版本准确、可验证、可回滚 |

## 4. 平台范围

| 平台 | 范围 |
| --- | --- |
| Windows 桌面端 | 主控端、被控端、登录、设备列表、远控、文件传输、录屏、设置、安装包 |
| Android 移动端 | 登录、设备列表、远控电脑、横屏全屏、支付、设置、APK 安装包 |
| 项目 API 服务 | 账号设备、最近连接、会员套餐、订单、支付回调、下载页 |
| RustDesk 服务器 | hbbs/hbbr 私有 ID 与中继服务 |
| 下载页 | Windows/Android 安装包下载、版本展示、邀请页跳转 |

不在当前版本范围内：

- iOS 商业发布。
- 浏览器端完整远控。
- 企业后台管理控制台。
- 多租户组织权限体系。

## 5. 核心业务流程

### 5.1 登录流程

1. 用户打开客户端。
2. 用户选择账号登录。
3. 客户端通过鲲穹登录体系获取登录 token 和用户信息。
4. 客户端保存本地登录状态、用户名、手机号、会员状态和项目 API token。
5. 登录后，客户端向项目 API 上报当前设备。
6. 退出登录时清理本地 token、用户信息和会员缓存。

验收标准：

- 登录后个人信息区显示用户名和手机号。
- 点击个人信息区域进入个人中心，不直接退出登录。
- 语言切换后设置页和其他主要页面应即时更新。
- 登录态应能被支付、设备列表、会员页复用，不应出现已登录但支付提示未登录的情况。

### 5.2 设备列表流程

设备列表分为以下标签：

| 标签 | 数据来源 | 展示规则 |
| --- | --- | --- |
| 最近连接 | 本地最近连接记录 | 显示本地全部最近连接；默认不展开；支持刷新动画 |
| 移动设备 | 当前账号登录过的移动设备 | 不显示本机；显示同账号其他手机/平板 |
| 桌面设备 | 当前账号登录过的桌面设备 | 不显示本机；显示同账号 Windows 等桌面设备 |

已取消：

- `常用设备` 标签不再展示。

设备列表功能要求：

1. 最近连接必须以本地记录为准，不应被服务器设备列表覆盖。
2. 移动设备和桌面设备必须以当前账号登录设备为准。
3. 本机不得出现在移动设备或桌面设备列表里。
4. 各标签为空时仍保留标签和空状态，不隐藏入口。
5. 每个标签支持展开和收起。
6. 最近连接刷新按钮点击后应有旋转动画。
7. 重命名设备后，最近连接和账号设备列表中的同一设备名称应保持一致。
8. 用户不应看到项目服务器内部信息；但用户自己配置的服务器地址可以继续展示和修改。

验收标准：

- 两台 Android 和一台 Windows 登录同账号后，任一设备上能看到除本机外的其他同账号设备。
- 最近连接数量稳定，不应出现进入页面后先显示 2 个、过一会变 4 个的抖动。
- 默认进入设备列表时，各分组不展开。

### 5.3 远程连接流程

1. 主控端输入远程设备 ID，或从设备列表/最近连接中选择设备。
2. 主控端通过密码、临时密码、确认授权等方式发起连接。
3. 被控端显示授权或根据密码策略自动连接。
4. 连接成功后主控端进入远程桌面页面。
5. 用户可使用鼠标、键盘、剪贴板、文件传输、录屏、画质设置等功能。

连接能力要求：

- 支持同局域网直连。
- 支持跨网络通过 hbbr 中继。
- 支持断线后的合理重连。
- 支持 Windows 服务模式，以便被控端无人值守和系统权限场景。
- 支持多显示器切换。
- 支持远程剪贴板。
- 支持文件传输。

### 5.4 Android 远控电脑体验

手机远控电脑的默认体验要求：

1. 连接电脑后默认横屏。
2. 连接电脑后默认全屏。
3. 底部蓝色长条不遮挡远程画面。
4. 常用操作入口放到右侧侧边栏。
5. 不显示“复制指纹”等不必要入口。
6. 鼠标操作逻辑使用应用原有映射逻辑，保持与操作提示一致。
7. 操作提示、手势说明和实际行为必须一致。
8. 自定义画质入口在 Android 端可点击并弹出设置框。

右侧侧边栏要求：

- 位置不能过低。
- 不遮挡核心画面。
- 支持展开必要操作。
- 图标和提示应与当前鼠标映射逻辑一致。

### 5.5 密码和安全设置

密码能力应与 Windows 端保持一致，至少支持三类密码/连接凭据：

1. 临时密码。
2. 固定密码。
3. 自定义或额外授权类密码。

要求：

- 三类密码应可查看当前状态。
- 用户应能修改密码。
- 密码配置变更后应立即生效。
- 被控端应能明确控制是否允许远程配置修改。

### 5.6 会员与画质权益

会员体系用于解锁更高远控体验和更多记录能力。

会员相关需求：

1. 客户端能获取当前用户会员状态和过期时间。
2. 会员状态应在桌面端和移动端同步。
3. 会员可使用更高分辨率、更高 FPS 和更高画质。
4. 非会员使用受限画质时，应看到明确的权益提示。
5. 长时间运行的远控窗口应能从本地同步最新会员状态，避免开通会员后仍锁定 720p/30 FPS。
6. 最近连接数量限制按会员状态区分：普通用户较少，会员更多。

验收标准：

- 开通会员后，不重启或尽量少重启客户端即可解锁对应画质/FPS。
- 画质、分辨率、FPS 设置在远控工具栏和设置页表现一致。
- 自定义画质点击后能正常弹出设置框。

### 5.7 支付流程

当前支付重点：

- Android 端支付宝优先使用提交表单/网页收银台流程。
- 微信支付入口当前可隐藏，直到 APP Pay 参数和商户配置完整可用。

支付需求：

1. 用户选择会员套餐后，点击按钮文案为“立即支付”。
2. 点击后若需要等待订单创建或拉起支付，应显示加载动画。
3. Android 支付宝不应先弹二维码页再拉起支付。
4. 用户取消支付宝后，回到应用应提示支付已取消，不应再次自动拉起支付宝。
5. 支付页不应出现多余的“等待支付确认”等干扰提示。
6. 支付完成后客户端轮询订单状态，成功后刷新会员权益。
7. 支付失败或取消应给出可理解的提示。

服务端支付需求：

- 项目 API 提供会员套餐接口。
- 项目 API 创建会员订单。
- 项目 API 查询订单状态。
- 项目 API 保存本地会员订单。
- 支持支付宝回调。
- 支持微信支付回调的服务端能力，但移动端入口可按版本策略隐藏。

已知支付策略：

- 支付宝 App Pay SDK 方式曾验证到 `isv.insufficient-isv-permissions`，属于支付宝开放平台/商户权限问题，不作为当前默认流程。
- 相关经验已保存为模板，后续如支付宝 APP 支付权限开通，可切回服务端签名 App Pay 方案。

### 5.8 录屏

远控录屏需求：

1. Windows 远控窗口提供开始录屏和停止录屏。
2. 用户可以选择录屏保存位置，或使用默认保存位置。
3. 录屏结束后应弹出提示，告知录屏文件路径。
4. 录屏文件应为常见播放器可播放格式，优先避免依赖系统额外 HEVC 扩展。
5. 无录屏权限或录屏失败时应给出清晰提示。

### 5.9 下载页和安装包发布

下载页需求：

1. 下载页提供 Windows 安装包下载按钮。
2. 下载页提供 Android 安装包下载按钮。
3. 页面底部展示 Windows 和 Android 版本号。
4. Android 页面版本号应显示最终实际可安装 APK 的 versionName/versionCode。
5. APK 使用 split-per-abi 构建时，下载页应显示 split 后 APK 的真实 versionCode。
6. 下载接口返回正确文件名和 Content-Disposition。
7. 下载接口提供 SHA256 校验信息。

当前线上 Android 版本要求：

- 包名：`com.carriez.flutter_hbb`
- versionName：`1.4.6`
- versionCode：`4067`
- 签名 MD5：`037ec2069ddb32f065626cbb2b17dba9`
- 下载地址：`https://remotelink.kunqiongai.com/kq-api/download/android`

注意：

- 测试服务器曾存在守护进程每 2 分钟杀死容器，`/kq-api/download` 和 `/kq-api/api/health` 短暂 502 属于测试服预期现象。
- `/kq-api/download/android` 由 Nginx 静态 alias 直接提供，通常不受 API 容器重启影响。

### 5.10 Windows 安装包与低误报发布

Windows 安装包要求：

1. 安装包品牌为鲲穹远程桌面。
2. 外层安装器和内层可执行文件/DLL 均应签名。
3. 发布前验证 Authenticode 签名状态。
4. 提供低误报安装包模式，减少杀毒软件误报。
5. 安装器不应静默执行高风险操作；服务安装、防火墙修复等尽量由用户主动触发。
6. 若安全软件误报，应记录 SHA256、签名信息和误报引擎，便于提交白名单。

低误报策略：

- 避免安装时静默执行 `netsh`、`sc.exe`、`taskkill`、服务安装等高敏行为。
- 保留桌面快捷方式、开始菜单、协议注册等基础安装能力。
- 服务启用、防火墙修复和推荐权限通过应用内明确按钮触发。

### 5.11 多语言

多语言需求：

1. 支持简体中文、繁体中文及其他 RustDesk 原有语言。
2. 选择语言后，当前页面和其他主要页面应立即更新。
3. 不允许繁体中文或其他语言界面混入未翻译英文，除非为产品名、技术名词或第三方固定名称。
4. 支付、会员、设备列表、设置页、远控工具栏等新增文案必须进入翻译资源。

验收标准：

- 切换到繁体中文后，不仅设置页，首页、设备页、会员页、远控页也应立即刷新。
- 简体中文、繁体中文均不出现明显漏翻。

## 6. 后台与数据要求

项目 API 需要维护以下数据：

| 数据 | 说明 |
| --- | --- |
| 用户映射 | 鲲穹账号 token 与项目本地用户记录 |
| 设备记录 | 当前账号登录过的设备、设备类型、本机识别信息、最后在线时间 |
| 最近连接 | 本地为主，服务端可保存同步记录，但不得覆盖本地展示逻辑 |
| 会员快照 | 用户会员状态、过期时间、套餐信息 |
| 会员订单 | 订单号、支付方式、支付状态、套餐、过期时间、原始支付信息 |
| 下载元数据 | Windows/Android 文件名、版本、SHA256、下载 URL |

项目 API 主要接口：

| 接口 | 方法 | 用途 |
| --- | --- | --- |
| `/api/health` | GET | 健康检查 |
| `/api/me` | GET | 当前用户和会员信息 |
| `/api/account-devices` | GET | 当前账号登录设备 |
| `/api/account-devices/current` | POST | 上报当前设备 |
| `/api/connection-history` | GET/POST/DELETE | 最近连接记录 |
| `/api/member/packages` | GET | 会员套餐 |
| `/api/member/orders` | POST | 创建会员订单 |
| `/api/member/orders/:orderNo` | GET | 查询会员订单 |
| `/api/alipay/notify` | POST | 支付宝回调 |
| `/api/wechat-pay/notify` | POST | 微信支付回调 |
| `/download` | GET | 下载页 |
| `/download/windows` | GET/HEAD | Windows 安装包 |
| `/download/android` | GET/HEAD | Android APK |

## 8. 非功能需求

### 8.1 稳定性

- 远控连接应能在普通家庭网络、公司网络和移动网络下稳定使用。
- API 短暂不可用时，客户端应有重试和友好提示。
- 下载接口应避免被单 IP 或全局并发拖垮。

### 8.2 安全性

- 不向普通用户展示项目内部服务器敏感信息。
- 用户自定义配置的服务器信息可以展示和修改。
- 支付私钥、服务器密钥、数据库密码不得写入公开仓库。
- Windows 安装包应完成内外层签名。
- 远控权限变更应由用户明确触发。

### 8.3 性能

- 设备列表进入后应尽快显示本地缓存，再刷新网络数据。
- 最近连接不应因多数据源合并造成反复跳动。
- 移动端远控操作延迟应尽量接近同类远控产品。

### 8.4 可维护性

- 下载版本号应由最终产物读取，避免人工硬编码错误。
- 支付流程应区分当前默认方案和备用方案。
- 服务器迁移应有明确替换点、验证步骤和回滚步骤。

## 9. 验收清单

### 9.1 登录与账号

- 用户可用鲲穹账号登录。
- 登录后显示用户名和手机号。
- 个人信息区域进入个人中心。
- 退出登录后本地状态清空。

### 9.2 设备列表

- 最近连接默认不展开。
- 最近连接显示本地全部记录。
- 移动设备显示同账号其他移动设备，不显示本机。
- 桌面设备显示同账号其他桌面设备，不显示本机。
- 空分组不隐藏。
- 刷新按钮有动画。
- 重命名在相关列表中同步。

### 9.3 远程控制

- Windows 到 Windows 可连接。
- Android 到 Windows 默认横屏全屏。
- 鼠标点击、拖拽、滚轮和键盘输入可用。
- 远控工具栏不遮挡核心画面。
- 剪贴板和文件传输可用。
- 录屏结束提示保存位置。

### 9.4 会员与支付

- 会员套餐可展示。
- 支付宝点击“立即支付”后有加载动画。
- 支付取消后不重复拉起。
- 支付成功后会员状态刷新。
- 微信支付入口按当前策略隐藏。
- 会员画质/FPS 权益立即生效或有明确刷新机制。

- 下载页显示 Windows 和 Android 下载按钮。

## 10. 已知风险和待确认项

| 风险/待确认项 | 当前状态 | 建议 |
| --- | --- | --- |
| 测试服务器周期性 502 | 守护进程每 2 分钟杀容器，当前视为预期 | 正式服务器不应保留该策略 |
| 支付宝 App Pay 权限 | 曾返回 `isv.insufficient-isv-permissions` | 当前使用网页表单方案；权限开通后再切 SDK |
| 微信支付拉起 | 服务端和客户端曾做过 APP Pay 方案，入口当前可隐藏 | 商户参数完整后再恢复入口 |
| 正式服务器迁移 | 已有迁移计划，需生产域名/IP 和回调 URL | 迁移前确认 hbbs key 策略 |
| 杀毒误报 | Windows 远控软件天然敏感 | 坚持内外层签名和低误报安装器 |
| 多语言漏翻 | 过去繁体和其他语言有英文残留 | 新增文案必须纳入翻译检查 |

## 11. 版本与发布要求

Android：

- 使用同一包名 `com.carriez.flutter_hbb`。
- 使用当前签名 MD5 `037ec2069ddb32f065626cbb2b17dba9`。
- versionCode 必须递增。
- 下载页展示最终 APK 的实际 versionCode。

Windows：

- 版本号按日期和构建号维护。
- 交付安装包前验证 SHA256。
- 交付安装包前验证 Authenticode。
- 发布到下载页前更新服务器下载元数据。

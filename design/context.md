# 设计上下文

目标：提升 Android 客户端的主流程体验，让远程连接、最近设备、被控端准备、账号、会员和设置都更清楚、更少误操作。

用户：中国 Android 用户，包括不太懂技术、通过 APK 安装、需要开启敏感权限的用户。

核心任务：
- 连接另一台设备。
- 查看最近连接和账号设备。
- 把本机准备成可被远程控制的设备。
- 管理账号、会员、画质、网络和安全设置。
- 在权限、设备列表、支付、连接异常时知道下一步该做什么。

约束：
- 现有 Flutter 移动端 UI。
- RustDesk 派生架构。
- Android APK 分发。
- 中文优先。
- 项目内置服务器信息默认不暴露；用户自定义配置仍允许展示和编辑。

成功标准：
- 每个页面 2 秒内能看出主操作。
- 权限、设备、账号状态能解释当前状态和下一步。
- 页面不应先显示误导性的 0、空状态或“加载中”。
- 普通用户看不到项目服务器内部信息。

本次范围：
- 审查 Android 移动端主页面和当前构建可进入的关键子流程。
- 输出问题优先级和修复建议。
- 本次不直接改 UI 代码。

非目标：
- Windows 桌面端改版。
- 后端协议调整。
- 完整 WCAG 认证。
- 品牌视觉重做。

假设：
- 这个窗口继续专注 Android。
- 简体中文是主要目标语言。
- 用户经常通过 APK 安装，所以 Android “受限设置”引导很重要。

风险：
- 支付、账号、设备列表部分依赖真实线上账号状态，本地模拟器无法完整覆盖所有生产状态。
- 无障碍结论是启发式审查，未做 TalkBack 全流程测试。
